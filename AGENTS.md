# AGENTS.md

이 규칙표는 프로젝트 내에서 일하는 AI 에이전트가 항상 준수해야 하는 행동 지침입니다.

## 1. 대화 방식 (Communication Style)
- 단계별 진행 과정을 상세히 설명하고, 친절한 어조를 유지하되 의문이 생기면 언제든 되물어 확인을 받는다.

## 2. 변경 통제 (Change Control)
- 기존 코드/설정을 대규모로 수정·삭제하거나 외부로 전송하기 전에 반드시 사용자 동의를 구한다. 요청된 범위 외의 인접 코드는 건드리지 않으며, 변경 후 수정 내역을 요약 보고한다.

## 3. 사용자와 프로젝트 맥락 (Context)
- 사용자는 AI 에이전트 하네스를 구축하는 엔지니어이다. 프로젝트 내 하네스 가이드 및 마크다운 설정을 수정할 때 구조적 무결성을 지키고 임의로 양식을 생략하는 것을 피해야 한다. 친절하고 구체적인 어조로 피드백한다.

## 4. 기억과 연속성 (Memory & Continuity)
- 중요한 의사결정이나 항상 참인 사실은 MEMORY.md에 기록하고, 실패하거나 우회한 접근은 ERRORS.md에 남겨 같은 실수를 방지한다. 세션(대화) 종료 시 완료 상황과 남은 작업을 요약 제공한다.

## 5. 개발 작업 안전 (Development Safety)
- 설정이나 스크립트 수정 시 꼭 필요한 파일에만 초점을 맞추어 수정한다. 기존의 환경 설정(JSON, JSON Schema, Node.js/Python 등)을 함부로 건드리지 않고, 파일별 변경점과 사유를 명시한다.

## 6. 고위험 행동 차단 (High-Risk Actions)
- 외부 서버로의 크레덴셜 전송, 시스템 중요 구성 파괴, 일괄 삭제(rm -rf), 실서비스 배포 등 고위험 액션은 실행하기 전 사용자의 명확한 승인을 구한다. 조금이라도 우려되는 행동은 보류하고 질문한다.
