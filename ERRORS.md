# ERRORS.md

이 파일은 에이전트가 이 프로젝트에서 과거에 겪었던 실패 사례와 삽질, 그리고 이를 우회하는 방법을 기록하여 동일한 실수의 반복을 방지합니다.

## 실패 사례 및 예방책 (Failures & Prevention)

### 1. 존재하지 않는 Git 명령어 사용 실패
- **실패 상황**: 작업 현황을 파악하기 위해 관성적으로 `git status`나 `git diff` 등의 명령어를 실행했으나, `.git` 리포지토리가 아니라서 에러가 발생함.
- **예방책**: Git 명령어 대신 일반 파일 탐색기 수준의 명령어(`ls`, `find`)나 Antigravity의 리스트 도구(`list_dir`)를 사용해야 합니다.

### 2. 하네스 마크다운 필수 스키마 누락
- **실패 상황**: 하네스 마크다운 파일(예: `SKILL.md`)을 작성하거나 수정할 때, 상단의 YAML Frontmatter 등 필수 양식을 임의로 변형하거나 누락하여 에러를 일으킴.
- **예방책**: 파일 구조를 변경할 때 반드시 템플릿 가이드라인을 참조하고, Frontmatter 영역은 손상되지 않도록 보존해야 합니다.

### 3. ArtifactMetadata 사용 시 TargetFile 경로 제약
- **실패 상황**: `write_to_file` 호출 시 `ArtifactMetadata`를 제공했으나, `TargetFile`이 아티팩트 보관용 격리 디렉토리가 아닌 로컬 소스 디렉토리(`docs/`)를 가리켜 파싱 예외 및 차단이 발생함.
- **예방책**: 프로젝트의 실제 코드 디렉토리나 문서 디렉토리(`docs/` 등)에 문서를 쓸 때는 `ArtifactMetadata` 필드를 통째로 제외하여 호출해야 하며, 사용자 브레인 보관 디렉토리에 전용 보고서 형태로 기록할 때만 아티팩트 메타데이터를 기입해야 합니다.

### 4. LLM JSON 반환 형식 불안정성으로 인한 파싱 폭사
- **실패 상황**: `responseMimeType: 'application/json'` 스펙을 전달했음에도 간혹 Gemini 모델이 백틱 코드 펜스(\`\`\`json ... \`\`\`)를 앞뒤로 두르고 JSON을 리턴하여 Node.js `JSON.parse` 시 구문 에러가 발생할 수 있음.
- **예방책**: 파싱을 집행하기 전 백틱 마크다운 문자열을 디펜시브하게 교정해 주는 유틸리티 함수(`cleanAndParseJson`)를 작성하여 감싼 후 파싱하도록 설계합니다.
